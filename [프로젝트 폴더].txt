[작업명]
glter 내 “내글보기” 기능 추가

[작업 목표]
기존 “사용자 배경 관리” 화면과 거의 동일한 UI/구조로 “사용자 내글 관리” 기능을 추가한다.

[핵심 원칙]
- 사용자 배경 관리 UI와 거의 똑같은 모양으로 만든다.
- 구조/버튼 배치/활성화 규칙도 최대한 동일하게 맞춘다.
- 차이는 콘텐츠만 다르다.
  - 사용자 배경 관리 = 이미지
  - 사용자 내글 관리 = 텍스트

[버튼명]
내글보기

[관리 화면 구성]
- 사용자 배경 관리와 같은 구조 사용
- 상단 버튼: 전체 선택, 삭제
- 하단 버튼: 추가, 적용
- 초기 진입 상태는 사용자 배경 관리와 동일하게 “추가만 활성화”
- 선택된 항목이 없으면 삭제/적용 비활성
- 선택된 항목이 있으면 삭제/적용 활성

[목록 UI]
- 사용자 배경 관리처럼 3열 그리드 구조 유지
- 이미지 썸네일 대신 텍스트 카드 사용
- 카드는 세로형 썸네일이 아니라 거의 정사각형에 가까운 모양으로 만든다
- 카드 안에는 글 미리보기 1~2줄만 표시하고 말줄임 처리
- 카드 선택/해제 가능
- 카드 탭 시 수정 화면 진입

[추가 기능]
- 추가 버튼 탭 시 글쓰기 화면 진입
- 글쓰기 화면은 메모지 느낌의 큰 텍스트 입력 UI
- 입력 방식은 타이핑, 붙여넣기만 지원
- 손글씨/직접 필기 입력 기능은 넣지 않는다

[글쓰기 화면 규칙]
- 세로 모드 / 가로 모드 모두 지원
- 기존 Passage 화면에서 사용 중인 텍스트 폭/좌우 여백 규칙을 그대로 적용
- 사용자가 작성할 때도 실제 표시 폭과 최대한 비슷한 느낌으로 입력되게 한다
- 줄바꿈은 자연스럽게 유지

[글쓰기 화면 버튼]
- 새 글 작성 시: 취소, 적용
- 수정 시: 삭제, 취소, 적용
- 수정 화면에서는 바로 삭제 가능해야 한다

[저장/상태 규칙]
- 저장된 글은 앱 내부 자동 저장
- 가져오기/내보내기 기능은 만들지 않는다
- 순서 변경 기능은 만들지 않는다
- 제목, 감정 태그, 중복 검사 기능은 만들지 않는다
- 같은 글 중복 저장 허용
- 새 글 저장 시 기본 active=true
- 선택된 글만 활성화 상태로 반영

[내글보기 동작]
- 내글보기만 선택 시: 사용자 글만 노출
- 기존 데이터 버튼과 함께 선택 시: 기존 데이터 + 사용자 글을 함께 후보군에 포함
- 내글보기가 선택되지 않으면 사용자 글은 후보군에 포함하지 않는다

[데이터 필드 최소 구성]
- id
- body
- active
- createdAt
- updatedAt
